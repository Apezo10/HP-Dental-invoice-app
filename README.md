# HP Dental Invoice App

A full-stack invoice and statement web app built for a dental technician workflow. The goal is to make billing feel as simple as using a spreadsheet while preserving clean historical records for invoices and statements.

## Overview

HP Dental Invoice App helps a dental lab create printable invoices and company-level statements. Customers and services are entered once, then reused from dropdowns so repeated data entry is minimized.

Statements are generated by company, which means one statement can include invoices for multiple customers under the same parent company.

## Features

- Email/password authentication with Supabase Auth
- Simple dashboard with two actions: Create Invoice and Generate Statement
- Customer management with reusable company information
- Company suggestions when adding customers
- Saved services with default prices
- Invoice rows with saved service, patient name, quantity, editable unit price, and calculated line total
- Historical invoice pricing, so old invoices do not change when service defaults change later
- Professional printable invoice PDFs
- Professional printable company statement PDFs
- Company-level statement generation across multiple customers
- Supabase row-level security policies
- Vercel-ready Next.js deployment

## Tech Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- Supabase database and authentication
- `@react-pdf/renderer` for printable PDFs
- Vercel deployment target

## Database

The Supabase schema includes:

- `customers`
- `services`
- `invoices`
- `invoice_items`
- `payments`

Run the schema from:

```text
supabase/schema.sql
```

If updating an older local database that does not have patient names yet, run:

```text
supabase/add-patient-name.sql
```

## Deployment

For a clean production deployment:

1. Create a fresh Supabase project.
2. Run `supabase/schema.sql`.
3. Deploy the app to Vercel.
4. Add the environment variables in Vercel project settings.

This keeps production data separate from local test data.
